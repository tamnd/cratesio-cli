First public release. Eight commands covering the full read surface of the
crates.io public API: search, single-crate metadata, version history, owners,
dependencies, top-downloads chart, categories, and keywords.

## Commands

| Command | What it does |
|---------|-------------|
| `crates search <query>` | Search crates.io; sort by downloads, alpha, new_crates, new_versions, or recent_downloads |
| `crates crate <name>` | Show metadata for a single crate |
| `crates versions <name>` | List all published versions (newest first) |
| `crates owners <name>` | List crate owners (users and teams) |
| `crates deps <name>` | Show dependencies of the latest version |
| `crates top` | Top downloaded crates; use `--page` to page through |
| `crates categories` | List all crates.io categories |
| `crates keywords` | List popular keywords |

## Output formats

Every command supports `-o table|json|jsonl|csv|tsv|url|raw`. The default is
`table` on a terminal and `jsonl` when piped.

```sh
crates search tokio -n 5 -o table
crates crate serde -o json
crates versions serde -n 10 -o csv
crates top --page 2 -o jsonl
```

## Install

```sh
# Homebrew
brew install tamnd/tap/crates

# Go install
go install github.com/tamnd/cratesio-cli/cmd/crates@latest

# Docker
docker run --rm ghcr.io/tamnd/crates:0.1.0 search serde -n 5
```

No authentication or API key required. The tool sets the User-Agent header that
crates.io requires by default.
